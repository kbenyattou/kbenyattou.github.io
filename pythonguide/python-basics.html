Everything in Python is an Object

Python is an object-oriented language. We'll formally define objects later but what this means is that everything in Python is an object: strings, integers, functions etc. Every object in python has an identity, type and value.

Identity

The identity of an object is the unique identifier for an object and is used by Python to retrieve the object. We can return the identity of an object with id(). Once created, the identity of an object never changes.

Type

The type of an object determines the operations that the object supports and defines the possible values for objects of said type. We can return the type of an object with type(). Once created, the type of an object also never changes.

Value

The value of some objects can change. This partitions objects into two groups: mutable and immutable.

---

Mutable and Immutable

An immutable object's value cannot be changed after it's created. Attempting to modify an immutable object's value will result in a copy being created.

Mutable objects' values can be changed after creation.

Object Type Examples
Immutable bool int float tuple str
Mutable list dict set

Variable Assignment

Variables are the primary way to store and refer to data in Python. We create variables in Python by putting a name to the left of an = sign and a value on the right e.g. x = 10.
