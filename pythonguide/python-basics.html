Variables

A variable in python is a reserved memory location to store data. When a variable is created, some space in memory is reserved. Unlike other languages, the declaration to reserve space automatically happens when a value is assigned to a variable in Python. The syntax for creating a variable is:

variable_name = initial_value

The data stored in memory can be of many types. Some standard data types are:

Numbers Integers, floats and complex numbers e.g. 2, 0.5, 2+5j
Strings Continuous sets of characters enclosed by quotation marks e.g. 'hello'
Lists Items separated by commas and enclosed by square brackets [1, 'hello', 10]
Tuples Similar to lists but the items are enclosed by parentheses and the items cannot be changed after creation e.g. (2,4)
Dictionaries A type of hash table i.e. a list of key:value pairs enclosed by curly brackets e.g. my_dict = {'name':'k', 'weight':50}

Unlike other languages, variable types in Python also need not be declared in your code.

Everything in Python is an Object

Python is an object-oriented language. We'll formally define objects later but what this means is that everything in Python is an object: strings, integers, functions etc. In this sense, every variable points to an object.

Every object in Python has an identity, type and value.

Identity

The identity of an object is the unique identifier for an object and is used by Python to retrieve the object from system memory. We can return the identity of an object with id(). Once created, the identity of an object never changes.

Type

The type of an object determines the operations that the object supports and defines the possible values for objects of said type. We can return the type of an object with type(). Once created, the type of an object also never changes.

Value

The value of some objects can change. This partitions objects into two groups: mutable and immutable.

---

Mutable and Immutable

A mutable object's content (value) can be changed without changing its identity.

An immutable object's value cannot be changed after it's created. Attempting to modify an immutable object's value will result in a new variable being created.

We can test the mutability of an object by using id(). For example, on my system:

s = 'abc'
id(s)
    > 4383489712
s += 'xyz'
print(s)
    > abcxyz
id(s)
    > 4385098032

x = [1,2]
id(x)
    > 4385029312
x.append(3)
print(x)
    > [1, 2, 3]
id(x)
    > 4385029312

We can see that the location in memory of the string variable s changes when we try to change its value. This means that strings are immutable. However, the identity of the list object x does not change when we change its value. This is because list objects are a mutable data type.

Object Type Examples
Immutable bool int float tuple str
Mutable list dict set
