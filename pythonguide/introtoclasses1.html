What is a Class?

If we were limited to strings, numbers and booleans, we'd have a tough time trying to model objects we see in the real world. To this end, we can write blocks of code that serve as blueprints for creating new data types. These blueprints are known as classes.

Classes are just specifications. We can't use classes themselves in our programs. Instead, we use classes to create objects (also known as instances). We use the word 'instantiate' to refer to the process by which an object is created from a class. By analogy with car manufacturing:

  • the class would be the design blueprints for a type of car,
  • the instantiation/creation step would be the building process,
  • and the object is the end-result, a car.

Classes allow us to logically group data and functions in a way that's easy to re-use and build upon. This is part of the reason why they're useful.

---

Creating and Instantiating Classes

We can create an empty class (without attributes and methods) by writing the following code. (Note that we conventionally capitalise class names so they are easy to identify.)

class ClassName:
    pass

To instantiate a class, we pass it as a variable. The syntax is reminiscent of a function call. (We can also pass instances into functions and do pretty much everything else we can do with basic data types. More on this later.)

Below we instantiate the same class twice.

instance1 = ClassName()
instance2 = ClassName()

Each instance created from a class is unique. For example, both instances above have different locations in system memory. Continuing the car analogy, if two identical cars are created from the same class, they are different objects despite being identically designed and built.

---

Attributes

Naturally, we'd like to imbue our objects with information that describes them. To this end, we can create variables that are particular to an object. Attributes refer to variables associated with a class.

Instance Variables

A great benefit of using classes is that each instance of a class can hold different kinds of data. This data is referred to as an instance variable. We can define instance variables as follows: 
instance1.name = 'First Last'
instance1.height = 170
instance1.weight = 60

This process would be quite laborious if we have many instances. To this end, we can use what is known in object oriented programming as a constructor. A constructor is an example of a function associated with a class. Such functions are called methods.
