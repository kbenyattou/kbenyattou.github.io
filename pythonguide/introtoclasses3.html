Dunder Methods

When a method name is surrounded by double underscores, we call it a dunder method. These methods are also called special or magic methods. We've already seen the __init__ method which is an example of a dunder method.

Two other commonly used dunder methods are: - __repr__ returns an unambiguous representation of the object. - __str__ returns a more user friendly representation of the object.

Such methods define how objects are printed and displayed e.g.

import datetime
today = datetime.date.today()
str(today)
> '2020-09-01'
repr(today)
> 'datetime.date(2020, 9, 1)'

We can define our own __repr__ to return a string that we can use to recreate an object in the same way as the __init__ method.

Familiar Dunder Methods

There are some common dunder methods that work in the background of basic code. For example, when adding two objects together a special method called __add__ is called on the objects:

  • For integers, 1+2 returns 3 but under the bonnet we have an __add__ method called on the integer object via int.__add__(1,2).
  • The same goes for string concatenation. In this case, str.__add__('a', 'b') returns the string 'ab'.

We can customise how addition works for custom objects by defining our very own add dunder method within the class.

class People:
    def __init__(self, givenName):
        self.name = givenName

    def __add__(self, other):
        return self.name + ' \
        and ' + other.name

This particular dunder add method takes in two instances self and other, and outputs their names together.

There are many examples of dunder methods in the standard library.

The len method is one such example and typing something like len('test') returns 4 and what's going on behind the scenes is that __len__ is being called on the str object 'test'.
